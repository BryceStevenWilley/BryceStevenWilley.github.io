title: A Better way to list langauges on your Resume
---

Somewhere I learned how to list langauges on my resume by "learning", "proficient", "expert", etc.
These categories never made a ton of sense to me to be honest. 

Better idea: what you are able to do with the language. 
Example: C++: can write an entire program/system from scratch, without an existing code base, following best practices and optimizations.
Python: can write a program from scratch, maybe not with best practices
Javascript: can add features to existing programs, triage and fix bugs
Prolog: can't do anything, haha.

Alternative idea: list the language, and the frameworks + specific features that you have experience. Downsides: a ton of features for most languages, impossible to list all.
